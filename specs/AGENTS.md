# Specs

Specs are created in this folder before implementing new features.

## Format

```
feature-name.md
```

## Sections

1. **Context** - Background and motivation
2. **Goal** - What the feature should achieve
3. **Study/Analysis** - Research findings and options evaluated
4. **Decisions** - Key architectural choices with rationale
5. **Proposed Spec** - Detailed behavior, component structure, state model
6. **Implementation Outline** - Ordered steps to build
7. **TODOs** - Actionable checklist for the implementation

## Guidelines

- Be concise; avoid repetition
- Include rationale for decisions
- Document out-of-scope items
- Keep an ordered implementation plan
- Specify acceptance criteria and scope boundaries
